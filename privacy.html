Privacy Policy

Last updated: 11 June 2026 · Effective: 11 June 2026

This Privacy Policy describes how Invoicing App ("the App") handles information when used to generate invoices through QuickBooks Online.

The App is developed and maintained by Kaushik Sangem (ABN 72764107589), an Australian sole trader ("the Developer"). The App is a private, single-tenant desktop application provided to one specific Australian business ("the User") for the User's own invoicing workflow. It is not distributed to the public, sold, or offered as a service to third parties.

1. Roles

  • The Developer (Kaushik Sangem) writes and maintains the software, holds the Intuit Developer Portal account, and manages the OAuth Client ID and Client Secret.
  • The User is the Australian business that installs and operates the App on its own Windows machine, owns the connected QuickBooks Online company, and owns all data created or processed through the App.

2. Information accessed

When the App is connected to the User's QuickBooks Online company, it accesses the following data through the official Intuit QuickBooks Online API:

  • Customer records — name, email, billing address, ABN, customer ID
  • Item / service records — name, code, unit price, tax codes
  • Invoices — invoice number, line items, amounts, GST, status, sent / paid timestamps
  • Company information — company name and realm ID (used to display the connected company)

The App does not access banking information, payroll data, employee records, transactions unrelated to invoicing, or any data outside the com.intuit.quickbooks.accounting OAuth scope.

3. Where data is stored

  • All data is stored locally on the User's Windows machine in %LOCALAPPDATA%\InvoicingApp\.
  • Local storage uses an SQLite database file (invoicing.db) protected by the operating system's standard user-account permissions.
  • OAuth access tokens and refresh tokens are encrypted at rest using Fernet (AES-128 in CBC mode with HMAC-SHA256 authentication) before being written to disk.
  • No App data is transmitted to or stored on any server other than Intuit's QuickBooks Online API endpoints.
  • The Developer does not have access to data stored on the User's machine. The Developer does not operate any backend, does not receive copies of the User's data, and cannot read the User's QuickBooks information.

4. How data is used

Data is used solely to:

  • Display dashboards, submission history, and invoice status to the User.
  • Create customers, items, and invoices in the User's QuickBooks Online company.
  • Trigger QuickBooks Online's standard "send invoice" email to the User's customers.

Data is not used for analytics, marketing, profiling, advertising, or training of machine-learning models.

5. Sharing and third parties

The App does not share data with any third party. The only external service contacted by the App is the Intuit QuickBooks Online API at *.api.intuit.com and OAuth endpoints at appcenter.intuit.com.

6. Data retention

  • Local data is retained for as long as the User continues to use the App.
  • Disconnecting the App (via Settings → Disconnect QuickBooks) immediately deletes the stored OAuth tokens from the User's machine.
  • Uninstalling the App does not automatically delete the local SQLite database; the User can manually delete the %LOCALAPPDATA%\InvoicingApp folder to remove all local data.
  • Data created in QuickBooks Online (customers, items, invoices) is retained according to QuickBooks Online's own data-retention policies and is unaffected by uninstalling the App.

7. Security measures

  • OAuth 2.0 is used for authentication; no passwords are ever requested or stored.
  • OAuth tokens are encrypted at rest using a per-machine encryption key.
  • All API traffic uses HTTPS / TLS 1.2 or higher.
  • The App runs only on the User's local machine and does not expose any network services other than a temporary localhost listener used during the OAuth authorisation callback.
  • The App applies the principle of least privilege by requesting only the com.intuit.quickbooks.accounting scope.

8. Privacy Act application

The Developer (Kaushik Sangem) is an Australian sole trader with annual turnover under A$3 million, and is therefore generally exempt from the Privacy Act 1988 (Cth). The Developer nevertheless follows the spirit of the Australian Privacy Principles voluntarily in relation to the App.

Because data is held only on the User's machine and the Developer has no access to it, any privacy queries from individuals whose data appears in the User's QuickBooks Online company should be directed to the User in the first instance. The User remains the controller of its own customer data.

9. Children's data

The App is for business use only. It does not knowingly collect data from children under 16.
